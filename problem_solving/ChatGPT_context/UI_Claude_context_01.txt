2. Key Technical Concepts:
- **Hytale server plugin** (Java 25, Gradle 9.2.1) with hexagonal architecture
- **ECS (Entity Component System)** — `Store<EntityStore>`, `Ref<EntityStore>`, `Holder<EntityStore>`, `ComponentType`, `RefChangeSystem`
- **`InteractiveCustomUIPage`** — Hytale's custom UI page system; server builds layout via `UICommandBuilder`, binds events via `UIEventBuilder`
- **`EventData`** — Hytale's BSON-based event payload for UI button callbacks; keys MUST start with uppercase
- **`KeyedCodec` / `BuilderCodec`** — BSON codec for deserializing event payloads from client; keys must match EventData keys
- **`AbstractPlayerCommand`** vs **`AbstractCommandCollection`** — command base classes; collection has subcommands, player command handles execution directly
- **`PlayerReadyEvent`** — fires when player's ClientReady packet is received; `playerRef.getHolder()` is null at this point (ECS not yet attached)
- **`IUiPresenter`** interface — contract for `showClassMenu`, `showTalentMenu`, `sendInfo`, `sendError`, `sendSuccess`
- **`@Deprecated(forRemoval=true)`** — requires `@SuppressWarnings("removal")` not `"deprecation"` to suppress compiler warning
