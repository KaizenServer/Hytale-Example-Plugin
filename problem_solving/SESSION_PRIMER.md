In this project we maintain a file at `problem_solving/CLAUDE_BIBLE.md`. Before doing anything, read it.

This file contains rules derived from real mistakes made in past sessions — not generic advice. Each rule has the exact context of what went wrong. You are expected to apply them actively, not just acknowledge them.

How we handle mistakes:
- If you make a wrong assumption, misdiagnose something, or state something that turns out to be false — it gets added to the bible as a new rule with full context.
- Rules are only added when something actually went wrong, not as precaution.
- The folder `problem_solving/` also contains incident folders for specific past problems with supporting documents. Check it if something feels familiar.

When in doubt about a state (file exists? process running? fix applied?): **check first, speak second.**
